## 修复 Docker 数据卷挂载问题

### 问题分析
当前 `docker-compose.yml` 使用了：
```yaml
volumes:
  - express-data:/app/backend
```
这会挂载**整个** `/app/backend` 目录，包括配置文件、依赖包等，导致：
- ❌ `update-config.json` 无法通过 git pull 更新
- ❌ `node_modules/` 占用大量空间
- ❌ 无法应用新的代码更改

### 解决方案

#### 1. 修改 docker-compose.yml
将挂载目录改为**只挂载数据库文件**：
```yaml
volumes:
  - express-data:/app/backend/database.sqlite
```

这样只有数据库文件会被持久化，其他文件都会随代码更新而更新。

#### 2. 在服务器上执行
```bash
cd /opt/stacks/XINDA_Express
docker-compose down
docker volume rm XINDA_express-express-data  # 删除旧数据卷
docker-compose up -d                        # 重新创建并启动
```

### 优点
- ✅ 数据库文件仍然持久化（发货记录、用户数据完全安全）
- ✅ 配置文件会随 git pull 更新
- ✅ 自动重新安装 node_modules
- ✅ 节省磁盘空间

### 风险
⚠️ 需要删除现有数据卷，但只会删除数据库文件，其他文件会从镜像重新创建
⚠️ 确保先执行 `git pull` 获取最新代码
