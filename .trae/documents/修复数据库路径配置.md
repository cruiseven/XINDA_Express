## 修复数据库路径配置

### 问题
当前后端代码从 `backend/database.sqlite` 读取，但用户希望：
- **物理路径**: `./data/database.sqlite`（项目根目录）
- **容器内路径**: `./data/database.sqlite`

### 解决方案

#### 1. 修改后端代码 `backend/db.js`
将数据库路径从 `path.join(__dirname, 'database.sqlite')` 改为 `./data/database.sqlite`

#### 2. 修改 docker-compose.yml
将挂载路径改为：
```yaml
volumes:
  - ./data/database.sqlite:./data/database.sqlite
```

#### 3. 更新 .gitignore
忽略 `data/database.sqlite`

### 最终结构
```
XINDA_express/
├── data/
│   └── database.sqlite    # 数据库文件（物理位置）
├── backend/
│   └── ...
├── docker-compose.yml
└── ...
```

### 服务器部署步骤
```bash
cd /opt/stacks/XINDA_Express
mkdir -p data  # 创建 data 目录
# 从备份复制 database.sqlite 到 data/ 目录
docker-compose up -d --build
```
