# 公司发货管理网站开发计划

## 技术栈选择
- **后端**: Node.js + Express（简单易用，适合快速开发）
- **数据库**: SQLite（轻量级，无需额外数据库服务，适合小型应用）
- **前端**: 原生HTML/CSS/JavaScript（简单直接，易于维护）
- **部署**: Docker（容器化部署）
- **端口**: 6000

## 数据库设计

### 1. 用户表 (users)
- id, username, password, created_at

### 2. 承运人表 (carriers)
- id, name, contact_person, phone, address, created_at

### 3. 发件人表 (senders)
- id, name, phone, address, created_at

### 4. 收货地址表 (addresses)
- id, recipient_name, recipient_phone, recipient_address, created_at

### 5. 发货记录表 (shipments)
- id, tracking_number, carrier_id, sender_id, address_id, 
  weight, amount, status, shipping_date, notes, created_at

## 功能模块

### 1. 用户认证
- 登录页面
- 密码验证（管理员：cruiseven/1qaz2wsx）
- Session管理

### 2. 发货记录管理
- 添加发货记录（选择承运人、发件人、收货地址）
- 编辑发货记录
- 删除发货记录
- 按月份查询
- 按承运人筛选
- 导出汇总数据

### 3. 基础数据管理
- 承运人管理（增删改查）
- 发件人管理（增删改查）
- 收货地址管理（增删改查）

## 开发步骤

1. **创建项目结构和基础文件**
   - 创建backend/server.js（主服务器）
   - 创建backend/database.js（数据库）
   - 创建package.json

2. **实现数据库模型**
   - 创建所有数据表
   - 初始化示例数据

3. **实现API接口**
   - 用户认证API
   - 发货记录API（增删改查、筛选、汇总）
   - 基础数据管理API

4. **实现前端界面**
   - 登录页面
   - 主界面（发货记录管理）
   - 基础数据管理页面
   - 筛选和汇总功能

5. **Docker配置**
   - 编写Dockerfile
   - 编写docker-compose.yml
   - 配置端口6000

6. **测试**
   - 本地测试所有功能
   - Docker环境测试

## 部署方式

```bash
# 开发环境运行
cd backend
npm install
npm start

# Docker部署
docker-compose up -d
```

访问地址：http://localhost:6000
